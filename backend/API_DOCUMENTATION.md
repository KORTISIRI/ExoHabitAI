## 1. Theory & Architecture

### How the Prediction Pipeline Works

The ExoHabitAI backend uses a **two-layer inference pipeline**:

#### Layer 1 — ML Model (Random Forest / XGBoost)
The trained model (`final_model.pkl`) is a scikit-learn `Pipeline` that internally applies:
1. `StandardScaler` on numeric features
2. `OneHotEncoder` on the spectral type categorical feature
3. A tuned `RandomForestClassifier` or `XGBClassifier` (whichever scored best)

It outputs a **class-probability** for `habitability = 1` (Habitable).

#### Layer 2 — Inference Preprocessing (matching training)
Before the model sees any input, the same preprocessing applied during training is replicated exactly:
1. **Physical-limit filtering** — values outside hard astrophysical ranges are replaced with dataset medians
2. **IQR + Z-Score outlier clipping** — values are clipped to the combined outlier bounds computed on training data
3. **Unit standardisation** — `st_lum` stored as log10(L/L_sun) is converted to linear luminosity
4. **Feature engineering** — `stellar_compatibility` and `orbital_stability` are recomputed from raw inputs
5. **Spectral type normalisation** — only the first letter is kept; unknown types default to the training mode (typically `G`)

All preprocessing parameters (`numeric_medians`, `outlier_bounds`, `spectral_type_mode`) are saved during training inside `models/preprocessing_profile.pkl` and loaded at API startup, ensuring **zero training–inference skew**.

#### Habitability Label Definition
The binary target used during training is defined by this deterministic rule:
```
habitable = 1  if and only if:
    pl_rade     ≤ 2.0          (rocky / super-Earth size)
    st_teff  ∈ [3900, 7200] K  (F, G, K, M main-sequence star)
    pl_eqt   ∈  [180, 330] K   (liquid-water temperature zone)
    pl_orbsmax ∈ [0.4, 2.2] AU (habitable-zone orbital distance)
    st_lum   ∈ [0.08, 2.5] L☉  (not too bright, not too dim)
```
The ML model learns to replicate this from the remaining features (mass, density, orbital period, metallicity, spectral type, radius, semi-major axis).

## 2. Setup & Installation

### Prerequisites
- Python 3.10+
- Virtual environment (recommended)

### Steps

```bash
# 1. Activate your virtual environment
python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # Linux / macOS

# 2. Install dependencies
pip install -r backend/requirements.txt

# 3. Train the model (if models/ are not yet present)
python notebooks/ML_Model_Training.py

# 4. Start the Flask API
python backend/app.py
```

The server starts at **http://localhost:5000** by default.

## 4. Model Loading

At startup, `app.py` calls `load_artifacts()` from `utils.py`, which loads:

| Artifact               | File                               | Purpose                               |
|------------------------|------------------------------------|---------------------------------------|
| Trained ML pipeline    | `models/final_model.pkl`           | Predict habitability probability      |
| Feature column list    | `models/feature_cols.pkl`          | Ensure correct column ordering        |
| Preprocessing profile  | `models/preprocessing_profile.pkl` | Replicate training-time preprocessing |

If any artifact is missing, the server still starts but `/predict` and `POST /rank` return **HTTP 503**.

### 5.3 POST /predict

Predicts the habitability of a **single exoplanet**.

**Request Headers:** `Content-Type: application/json`

**Request Body**

| Field         | Alias                    | Type   | Required | Description                        | Unit         |
|---------------|--------------------------|--------|----------|------------------------------------|--------------|
| `pl_orbper`   | `orbital_period`         | float  | YES      | Orbital period                     | days         |
| `pl_orbsmax`  | `semi_major_axis`        | float  | YES      | Semi-major axis                    | AU           |
| `st_teff`     | `star_temperature`       | float  | YES      | Star effective temperature         | K            |
| `st_met`      | `star_metallicity`       | float  | YES      | Star metallicity [Fe/H]            | dex          |
| `st_spectype` | `spectral_type`          | string | YES      | Spectral type (first letter used)  | —            |
| `pl_rade`     | `planet_radius`          | float  | no       | Planet radius                      | Earth radii  |
| `pl_bmasse`   | `planet_mass`            | float  | no       | Planet mass                        | Earth masses |
| `pl_eqt`      | `equilibrium_temperature`| float  | no       | Equilibrium temperature            | K            |
| `pl_dens`     | `planet_density`         | float  | no       | Planet bulk density                | g/cm³        |
| `st_lum`      | `star_luminosity_log`    | float  | no       | Stellar luminosity log10(L/L_sun)  | —            |
| `pl_name`     | `planet_name`            | string | no       | Label (cosmetic only)              | —            |
| `hostname`    | `host_name`              | string | no       | Host star label (cosmetic only)    | —            |

**Example Request**
```json
{
  "pl_orbper":   365.25,
  "pl_orbsmax":  1.0,
  "st_teff":     5778,
  "st_met":      0.02,
  "st_spectype": "G",
  "pl_rade":     1.0,
  "pl_bmasse":   1.0,
  "pl_eqt":      288.0,
  "pl_dens":     5.51,
  "st_lum":      0.0,
  "pl_name":     "Earth",
  "hostname":    "Sol"
}
```

**Response (200 OK)**
```json
{
  "status":                   "success",
  "planet_name":              "Earth",
  "host_name":                "Sol",
  "prediction":               1,
  "label":                    "Potentially Habitable",
  "habitability_status":      "Habitable",
  "habitability_probability": 0.872314,
  "habitability_score":       0.872314,
  "confidence":               "87.23%",
  "threshold":                0.5,
  "status_message":           "Prediction generated successfully."
}
```

**Response Fields**

| Field                      | Type   | Description                                  |
|----------------------------|--------|----------------------------------------------|
| `prediction`               | int    | `1` = Habitable, `0` = Non-Habitable         |
| `label`                    | string | Human-readable prediction label              |
| `habitability_status`      | string | `"Habitable"` or `"Not Habitable"`           |
| `habitability_probability` | float  | Model confidence score (0.0 to 1.0)          |
| `habitability_score`       | float  | Alias of probability (for frontend use)      |
| `confidence`               | string | Probability formatted as percentage          |
| `threshold`                | float  | Decision boundary (always `0.5`)             |

### 5.7 POST /train-and-predict

Used by the **Raw Analysis** panel. Parses an uploaded or pasted dataset, runs the trained model on every row, and returns per-row habitability predictions together with training-style summary statistics.

**Accepted formats:**
- Multipart file upload — field name `file`, type `.csv` or `.json`
- Raw CSV text body (Content-Type: `text/csv`)
- JSON array (Content-Type: `application/json`)

**Recognised human-readable column names** (in addition to all NASA aliases):

| Column in your file       | Maps to     | Description                     |
|---------------------------|-------------|---------------------------------|
| `planet_radius`           | `pl_rade`   | Planet radius (Earth radii)     |
| `planet_mass`             | `pl_bmasse` | Planet mass (Earth masses)      |
| `orbital_period`          | `pl_orbper` | Orbital period (days)           |
| `semi_major_axis`         | `pl_orbsmax`| Semi-major axis (AU)            |
| `surface_temp`            | `pl_eqt`    | Equilibrium temperature (K)     |
| `star_temperature`        | `st_teff`   | Stellar effective temperature   |
| `star_metallicity`        | `st_met`    | Metallicity [Fe/H] dex          |
| `luminosity`              | `st_lum`    | Stellar luminosity log10(L/L☉) |
| `spectral_type`           | `st_spectype`| Spectral class letter           |
| `habitability_label`      | *(target)*  | Actual label — enables accuracy |

**Example CSV (paste in UI):**
```csv
planet_radius,planet_mass,orbital_period,semi_major_axis,surface_temp,star_temperature,star_metallicity,luminosity,spectral_type,habitability_label
1.1,1.0,365,1.0,288,5778,0.0,0.0,G,1
0.5,0.3,180,0.7,320,4800,-0.2,-0.3,K,1
12.0,300,10,0.05,900,25000,0.2,3.0,B,0
```

**Response (200 OK)**
```json
{
  "status": "success",
  "training": {
    "samples_used": 3,
    "train_size":   2,
    "test_size":    1,
    "accuracy":     1.0,
    "accuracy_pct": "100.00%",
    "labeled_rows": 3,
    "note": null
  },
  "predictions": [
    {
      "row_index": 0,
      "prediction": 1,
      "label": "Potentially Habitable",
      "habitability_score": 0.814,
      "confidence": "81.40%",
      "actual_label": 1
    }
```

> **Note:** `accuracy` reflects how well the *existing* model's predictions match the `habitability_label` values in your uploaded file — the model itself is not retrained.

## 7. Response Structure

All responses follow a consistent envelope:

**Success**
```json
{ "status": "success", "<result_fields>": "..." }
```

**Error**
```json
{
  "status":  "error",
  "message": "Human-readable description",
  "error":   "Same as message",
  "details": ["list of specific field errors"]
}
```

**HTTP Status Codes**

| Code | Meaning                                                       |
|------|---------------------------------------------------------------|
| 200  | Request successful                                           |
| 400  | Bad request — missing/invalid fields or wrong payload format |
| 404  | Ranked CSV not found — run ML_Model_Training.py first        |
| 503  | Model not loaded — run ML_Model_Training.py first            |

## 8. Input Validation & Error Handling

The API validates every request through `normalize_payload()` in `exohabit_pipeline.py`:

1. **Required field check** — `pl_orbper`, `pl_orbsmax`, `st_teff`, `st_met`, `st_spectype` must be present and non-empty.
2. **Numeric parsing** — All numeric fields cast to `float`; non-numeric values return 400.
3. **Physical limit filtering** — Values outside hard astrophysical bounds are silently replaced by training-set medians.
4. **Spectral type normalisation** — Only the first character is used; unknown types fall back to the training-set mode (`G`).

**Example Validation Error**
```json
{
  "status":  "error",
  "message": "Invalid input payload.",
  "details": [
    "Missing required field: pl_orbper",
    "st_teff must be a number"
  ]
}
```

## 9. Testing the API

### Browser
Navigate to `http://localhost:5000/` or `http://localhost:5000/health`.

```bash
    "st_teff": 5778, "st_met": 0.0,
    "st_spectype": "G", "pl_rade": 1.0,
    "pl_eqt": 288.0, "pl_name": "Earth"
```

```bash
curl "http://localhost:5000/rank?top=10"
```
